# 大热Key分析

数据库自治中心内置 Redis 大热Key分析功能，支持实时统计超过阈值的大Key与热Key，精准呈现 Key 的数据类型、元素个数及访问频次等核心数据。该功能为 Redis 性能优化提供科学依据，助力规避因 Key 分布倾斜引发的内存不足、访问热点等问题，保障实例稳定运行。


## 功能说明
- 大热Key分析需要 Redis 最新版本支持，旧版本实例进入功能页面时将提示进行版本升级，请按照页面引导完成升级后再使用。
- 支持查看实时数据与历史数据，历史数据查询时间范围为3天内，且单次查询跨度不超过3小时。
- 大Key阈值与热Key阈值可在统计策略中自定义配置，大Key默认阈值为100（元素个数），热Key默认阈值为500（访问频次）。
- 实时数据页面支持设置自动刷新间隔，可选不自动刷新、30秒、1分钟或3分钟。
- 分布式实例支持按分片筛选，查看各分片的大热Key分布情况。
- 大热Key统计仅覆盖客户端实际操作过的 Key，不会实时扫描数据库中的所有 Key，长时间未被访问的 Key **可能不会纳入统计**。
- **实例发生重启或主备切换后，已统计的大热Key数据将被清空。**
- 大Key统计数据存在一定延迟，非实时更新，请以页面展示时间为准。
- **开启大热Key统计功能后，在极端情况下可能对实例性能产生轻微影响。**


## 具体操作

在实例管理列表中，点击目标实例操作列的"..."，选择"大热key分析"进入操作页面。
![image](/images/bighot_key_analysis_0.png)

进入页面后默认展示实时数据，"大Key Top18"标签页展示元素个数超过阈值的大Key，包含 DB、数据类型、元素个数及 Key 名称信息。
![image](/images/bighot_key_analysis_1.png)

切换至"热Key Top50"标签页，可查看访问频次超过阈值的热Key，包含 DB、数据类型、访问频次及 Key 名称信息。
![image](/images/bighot_key_analysis_2.png)

点击刷新间隔下拉菜单，可设置实时数据的自动刷新频率，可选"不自动刷新"、"30秒"、"1分钟"或"3分钟"。
![image](/images/bighot_key_analysis_3.png)

切换至"历史数据"标签页，通过时间范围选择器查询历史大热Key数据，支持时间范围为3天内，跨度为3小时内。
![image](/images/bighot_key_analysis_4.png)

对于分布式实例，可通过右上角的"集群"下拉菜单按分片进行筛选，列表中会额外展示分片ID信息。
![image](/images/bighot_key_analysis_5.png)

页面右侧"统计策略"面板可修改大热Key的统计开关及阈值，点击"修改"按钮进行配置。


## 常见问题
### 为什么大热Key分析页面没有数据？
请检查右侧"统计策略"面板中大Key统计或热Key统计的开关是否已开启。统计功能关闭时不会记录任何大热Key数据。
### 为什么进入大热Key分析页面提示需要版本升级？
大热Key分析功能需要 Redis 最新版本支持，当前实例版本过旧，请按照页面引导完成版本升级后再使用该功能。
### 历史数据查询范围有什么限制？
历史数据支持查询3天内的记录，且单次查询的时间跨度不能超过3小时。
### 分布式实例如何查看各分片的大热Key？
进入大热Key分析页面后，通过右上角的"集群"下拉菜单选择对应分片，即可查看该分片的大热Key数据。
### 为什么有些Key访问量很高但没有出现在热Key列表中？
大热Key统计仅覆盖功能开启后客户端操作过的 Key，开启前的历史访问记录不会被追溯。如确认功能已开启，请检查该 Key 的访问频次是否达到热Key阈值。
### 为什么String类型的Key出现在大Key列表中？
对于String类型，大Key判断基于其value的长度而非子元素数量，value长度超过阈值时即会被列为大Key。
